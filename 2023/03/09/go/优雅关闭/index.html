go 优雅关闭

发表于 2023-03-09 分类于 go 源码阅读

优雅关闭仅支持 Linux 平台，利用捕获信号量 SIGINT 和 SIGTERM 后执行函数清理，睡眠一段时间后强制关闭。

go http shutdown 和 close 的区别？
https://www.dawxy.com/article/golang-tcp-shutdown-%E5%92%8C-close-%E7%9A%84%E5%8C%BA%E5%88%AB/

close 是直接关闭，shutdown 是优雅关闭。

优雅关闭的一些思考

即我们应用的server应该是最早被关闭的，接着是一些我们注册在 pkg 中的依赖，最后是基础依赖（如日志）

go-zero 对此做了一个设计，我们可以进行参考。

首先，他分成了 windows 和 linux 两种 build 文件，在 windows 里的优雅关闭什么都不做，我们直接看 Linux 版本的实现。

可以看到，他分成了两批进行关闭，观察源码，可以发现他将 HTTP、HTTPS、rpcserver、KeepAlive 服务的通知放到了 wrapUpListeners 中，剩下的 cache清理、metric 等基础服务都是注册到 shutdownListeners里的。

但是他把 ServiceGroup（用来注册除了 HTTP，rpc 服务的其他服务，如定时任务，MQ消费者等）里的服务关闭注册到了 shutdownListeners 里，这里不是没有很明白。

对于 server 为 HTTP 的应用，我们睡眠尽可能长的时间（保证 99的程序是执行完的），我们不可能等所有的长尾连接都运行完的。

市面上有方法是启用子进程，父进程继续处理。但是这样存在端口占用的情况。或许我们也可以启用一个新的进程，父进程把 HTTP 关了，但是继续处理未处理完的请求。如果请求是写入请求，那么这么做是有意义的，但如果请求是查询的就没必要。所以应该在连接数少的时间段进行优雅关闭。

同时，我们应该避免其他请求接入我们的 HTTP 服务，我们使用官方提供的 Shutdown 函数即可。
